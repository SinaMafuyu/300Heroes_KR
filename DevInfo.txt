# 패치파일 생성기

1. 패치를 하기 위해서는 .patch2 확장자의 패치 파일을 제작해야 하는데, 프로그램을 종료하기 전까진
실제 파일에 적용이 안된다. 반드시 적용 버튼을 누른 뒤, 프로그램을 종료해주어야 한다.

2. 과거에 사용하던 .patch파일을 patch2파일 제작하는곳에 끌어다 놓으면 patch2로 변환해준다.

3. 기존에 있던 위치에 바꿔넣는 형식이 아닌, 추가로 삽입하고, 링크주소를 변경하는 방식이다. 여러번 포함하면 용량만 늘어난다.



# .dat

1. 한글 패치를 하기 위해 .dat 파일을 수정할 경우 반드시 총 byte수가 기존과 일치해야한다.

2. 0x0a는 개행문자를 표현하는 바이트다. 이 바이트가 기존 .dat과 위치가 항상 동일해야한다. (더 많아도, 적어도 안된다.)

2-1. Notepad++도 결코 완벽하지 못하다. HEX-Editer를 동시 이용하면서 수정할 필요가 있다.

2-2. Notepad++의 HEX-Editer 확장은 버그가 있다. 외부 Editer 사용을 권장한다.

3. 위 사항을 불만족시 게임 참여가 불가능하다 (오류 발생)



# string_client_c.dat 작성규칙

1. ID 항목은0axx0axx로 시작. 12로 종료 (16진수 HEX 데이터)

2. String 값은 ID 항목 다음 비트를 건너뛰고 그 다음 비트부터 존재. 종료 표시자는 없으나 다음 ID항목의 시작값인 0a로 인식 가능.



# font\msyh.ttf

1. 폰트는 한글 Locale, 한글 폰트이어야 한글이 출력된다.

2. 클라이언트가 ANSI이기 때문에 한글만 출력되고, 중국어는 이상하게 깨지게 된다.

3. 한글 폰트일 경우 Locale을 중국어로 변경하면 모든 글씨가 깨지게 된다. (중국어 폰트가 없기 때문, Locale때문에 한글도 깨짐)



# .x

1. offzip을 이용해 2개의 파일로 unzip이 가능하다. 다만 생성된 파일을 어떻게 해야 Edit이 가능한지 모르겠다.

1-1. 명령어: offzip -z 15 -a [파일명].x "출력폴더경로" 0

2. 바이두에서 아무리 찾아봐도 답이 없다. 다들 포기한 듯. 포기하는게 편하다라는 답변이 많다.
